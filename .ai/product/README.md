# Product Knowledge Base

## Purpose

This directory contains the complete product knowledge base for the Landing Page A/B Planner.

It serves as the single source of truth for all product-related decisions.

Every AI agent and every contributor must reference this documentation before making product, business, UX, engineering, or roadmap decisions.

The goal is to ensure consistency, reduce ambiguity, and prevent conflicting implementations as the product evolves.

---

# Documentation Principles

This knowledge base follows five principles:

1. Single Source of Truth
2. Customer-Centric Decision Making
3. MVP First
4. Scalability by Design
5. Documentation Before Implementation

If a decision is not documented here, it should not be assumed.

---

# AI Usage Rules

AI agents must:

- Read relevant product documents before proposing features.
- Never invent business rules.
- Never expand the MVP scope without approval.
- Respect documented pricing and feature gates.
- Use the glossary for consistent terminology.
- Reference documented product philosophy when making recommendations.

If documentation is missing or ambiguous, request clarification instead of making assumptions.

# Maintenance Policy

Product documentation should be updated whenever:

- A feature is added or removed.
- Business rules change.
- Pricing changes.
- User personas evolve.
- Roadmap priorities shift.
- Success metrics are updated.

Documentation updates are considered part of the Definition of Done for any feature affecting product behavior.
